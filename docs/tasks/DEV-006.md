# DEV-006

## Title

Document Upload

---

## Goal

Добавить возможность загрузки документов в активный проект.

На данном этапе документы только сохраняются в проекте. Анализ содержимого документа не выполняется.

---

## User Story

Как системный аналитик,

я хочу загрузить документ в проект,

чтобы использовать его позже для AI-анализа, генерации требований и построения диаграмм.

---

## Description

Пользователь открывает Workspace проекта.

Нажимает кнопку:

📄 Загрузить документ

После этого бот ожидает файл.

После получения файла документ сохраняется в активный проект.

---

## Functional Requirements

### FR-1

Workspace содержит кнопку:

📄 Загрузить документ

---

### FR-2

После нажатия бот переходит в режим ожидания документа.

---

### FR-3

Поддерживаются форматы:

* PDF
* DOCX
* TXT
* MD

Если пользователь отправляет неподдерживаемый формат, бот отвечает:

❌ Данный формат пока не поддерживается.

---

### FR-4

Если активный проект отсутствует, бот отвечает:

❌ Сначала выберите проект.

---

### FR-5

После успешной загрузки документ сохраняется в проекте.

---

### FR-6

После загрузки бот отображает:

✅ Документ успешно загружен.

Имя файла.

Тип файла.

Количество документов в проекте.

---

## Non Functional Requirements

* Handler не содержит бизнес-логики.
* Используется Clean Architecture.
* Handler работает только через Service.
* Storage не вызывается напрямую из Handler.

---

## Acceptance Criteria

* Пользователь может загрузить PDF.
* Пользователь может загрузить DOCX.
* Пользователь может загрузить TXT.
* Пользователь может загрузить MD.
* Документ сохраняется в активном проекте.
* При отсутствии активного проекта загрузка невозможна.
* При неподдерживаемом формате отображается ошибка.

---

## Definition of Done

* Загрузка документа работает.
* Документ сохраняется.
* Код проходит compile.
* Выполнен smoke test.
* Архитектура не нарушена.

---

## Technical Notes

Добавить модель:

Document

Поля:

* id
* file_name
* file_type
* telegram_file_id
* uploaded_at

Добавить DocumentService.

DocumentService отвечает за:

* upload_document()
* list_documents()

ProjectService не должен содержать логику управления документами.

---

## Out of Scope

Не реализовывать:

* AI-анализ документа;
* OCR;
* чтение PDF;
* извлечение текста;
* генерацию требований;
* поиск по документу.

Только загрузка и хранение документа.
