# DEV-005

## Title

Project Workspace

---

## Goal

Добавить рабочее пространство проекта.

После выбора проекта пользователь должен перейти в карточку проекта, где будут доступны все дальнейшие действия.

---

## User Story

Как системный аналитик,

я хочу открыть существующий проект,

чтобы работать с документами, требованиями и диаграммами именно в его контексте.

---

## Description

После выбора проекта из списка открывается Workspace проекта.

Workspace отображает основную информацию о проекте и меню дальнейших действий.

На данном этапе большинство функций могут быть представлены как заглушки.

---

## Functional Requirements

### FR-1

После выбора проекта бот открывает Workspace.

---

### FR-2

Бот сохраняет выбранный проект как активный.

Использовать:

context.user_data["active_project_id"]

---

### FR-3

Workspace отображает:

- название проекта;
- статус;
- количество документов.

---

### FR-4

Workspace содержит действия:

📄 Загрузить BRD

🧠 AI Анализ

📝 Требования

📊 Диаграммы

⚙️ Настройки проекта

---

### FR-5

Все действия, кроме открытия Workspace, пока отображают сообщение:

🚧 Функция находится в разработке.

---

## Non Functional Requirements

- Использовать ProjectService.
- Handler не содержит бизнес-логики.
- Не обращаться напрямую к Storage.
- Соблюдать Clean Architecture.

---

## Acceptance Criteria

- Проект можно открыть.
- active_project_id сохраняется.
- Workspace отображается корректно.
- Handler использует только ProjectService.
- Проект запускается без ошибок.

---

## Definition of Done

- Workspace реализован.
- Код проходит compile.
- Выполнен smoke test.
- Архитектура не нарушена.

---

## Technical Notes

Добавить в ProjectService:

- get_project(project_id)

Использовать существующую модель Project.

Не изменять Storage без необходимости.

---

## Out of Scope

Не реализовывать:

- загрузку BRD;
- AI-анализ;
- генерацию требований;
- диаграммы.

Только Workspace.
